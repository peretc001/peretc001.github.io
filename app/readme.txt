Описание функционала БД

1. БД должна обеспечивать одновременный доступ пользователей к содержащимся в ней данным
через сеть Интернет при наличии уч?тной записи (логин, пароль).
2. В БД должна быть предусмотрена возможность добавления новой организации, редактирования
информации об организации и удаления организации.
3. В БД должна быть предусмотрена возможность поиска нужной организации по ключевым полям.
4. В БД должна быть предусмотрена возможность устанавливать контрольные даты и время (далее
– контрольные точки) и сортировать организации в соответствии с этими контрольными точками.

Структура БД
1. Главное окно (скрин 1)
Главное окно содержит два основных блока:
1. поиск организаций по заданным критериям, представленное полями:
1.1 строка поиска
1.2 критерии поиска
1.3 результаты поиска (слева)

2. Организации на контроле (справа)
В строке поиска можно указать часть искомой информации, указать в критериях поиска, по какому
полю будет осуществляться поиск и нажать найти. Результаты найденных совпадений в алфавитном
порядке отобразятся в поле результатов поиска.
В блоке «Организации на контроле» отображаются организации, поставленные менеджером на
контроль (только свои организации). Данный блок представлен следующими полями:
- наименование организации;
- заметка (выводится последняя строка поля «Результаты переговоров»)
- Дата (выводится «Контрольная дата»)
- Время (выводится «Контрольное время»)

Сортировка организаций осуществляется по дате, а при указании времени, дополнительно и по
нему. Если время не задано, то организации с той же контрольной датой отображаются последними
В блоке выводится список организаций, у которых поле «Контрольная дата» не пустое, отвечающих
критерию: Контрольная Дата ? Текущая Дата +7.

Переход к карточке предприятия осуществляется кликом по организации в результатах
поиска, либо по организации в списке организаций на контроле.

2. Карточка предприятия (скрин 2) все поля 

Поля свойств организации:
1. Наименование
2. Описание деятенльности
3. Регион (Поле "Регион" будет содержать список субъектов РФ)
4.  Адрес
5. ИНН 
6. Ответственный менеджер
7. Контактная информация
8. Результату переговоров
9.  Контрольная дата и время

Роли пользователей:
1. Гость
Права:
- осуществлять поиск организаций по критериям;
- просматривать карточки предприятий без возможности редактирования информации в них;
- видит организации всех менеджеров в блоке «Организации на контроле».

2. Менеджер
Права:
- добавлять новые организации;
- осуществлять поиск организаций по критериям;
- в блоке «Организации на контроле» видит только свои организации
- редактировать поля карточек предприятий (только своих), за исключением поля «Менеджер».

3. Администратор
Права:
- добавлять новые организации;
- удалять организации;
- осуществлять поиск организаций по критериям;
- редактировать любые карточки предприятий, в том числе менять менеджера;
- видит организации всех менеджеров в блоке «Организации на контроле».


Доступ к БД будет только для авторизованных пользователей, в т.ч. с типом учётной записи "Гость"
Регистрацией пользователей и назначением им ролей будет заниматься администратор.
